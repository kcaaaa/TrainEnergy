## 1. 设计概述

### 1.1 设计目标
高铁站节能降耗管理系统的UI设计以简洁、高效、直观为核心目标，旨在为用户提供清晰的能耗数据可视化展示和便捷的系统操作体验。设计遵循Element UI的设计规范，结合高铁站运营管理的特点，打造专业、现代的管理系统界面。

### 1.2 设计原则
- **简洁性**：界面简洁明了，避免冗余信息，突出核心数据和功能
- **一致性**：遵循统一的设计语言和交互规范，确保系统各模块风格一致
- **可用性**：操作流程直观易懂，减少用户学习成本
- **响应式**：适配不同屏幕尺寸，支持桌面端、平板端和移动端访问
- **可视化**：采用丰富的图表展示能耗数据，提高数据可读性

### 1.3 适用平台
- 桌面端：Windows、macOS、Linux
- 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 分辨率：支持1920×1080、1440×900、1366×768等主流分辨率

## 4. 布局系统

- **经典三栏布局**：

## 5. 组件设计

### 5.1 通用组件

#### 5.1.1 按钮
- **主要按钮**：使用主色调（#409EFF），用于重要操作
- **次要按钮**：使用白色背景+边框，用于次要操作
- **成功按钮**：使用成功色（#67c23a），用于操作成功场景
- **警告按钮**：使用警告色（#e6a23c），用于需要注意的操作
- **危险按钮**：使用危险色（#f56c6c），用于高风险操作（如删除）

- **按钮尺寸**：
  - 大型按钮：40px高度，用于重要操作
  - 中型按钮：32px高度，用于常规操作
  - 小型按钮：28px高度，用于次要操作
  - 超小型按钮：24px高度，用于紧凑场景

#### 5.1.2 输入框
- 使用Element UI的Input组件，支持文本、密码、数字等类型
- 输入框高度：32px（默认）
- 输入框宽度：根据内容自适应，默认100%
- 支持前缀图标、后缀图标、清除按钮等功能

#### 5.1.3 表格
- 使用Element UI的Table组件，支持分页、排序、筛选、导出等功能
- 表格行高：48px（默认）
- 表头背景色：#f5f7fa
- 表格边框：1px实线，颜色#dcdfe6
- 支持行选中、行展开、固定列等功能

#### 5.1.4 卡片
- 使用Element UI的Card组件，用于内容区块划分
- 卡片阴影：0 2px 12px 0 rgba(0, 0, 0, 0.1)
- 卡片圆角：4px
- 卡片内边距：20px

#### 5.1.5 表单
- 使用Element UI的Form组件，支持表单验证、布局调整等功能
- 表单布局：默认垂直布局，可根据需要调整为水平布局
- 表单项间距：16px
- 标签宽度：120px（水平布局时）

### 5.2 数据可视化组件

#### 5.2.1 折线图
- 使用ECharts的Line组件，用于展示能耗趋势
- 支持多条数据线、图例、tooltip、数据标记等功能
- 图表高度：300px（默认），可根据需要调整

#### 5.2.2 柱状图
- 使用ECharts的Bar组件，用于展示能耗对比、排名等
- 支持堆叠柱状图、分组柱状图、3D柱状图等
- 图表高度：300px（默认），可根据需要调整

#### 5.2.3 饼图/环形图
- 使用ECharts的Pie组件，用于展示能耗占比
- 支持普通饼图、环形图、南丁格尔图等
- 图表直径：300px（默认），可根据需要调整

#### 5.2.4 仪表盘
- 使用ECharts的Gauge组件，用于展示实时能耗、负载率等关键指标
- 支持自定义仪表盘颜色、刻度、指针样式等
- 图表直径：200px（默认），可根据需要调整

### 5.3 导航组件

#### 5.3.1 左侧导航栏
- 固定宽度：200px（展开）/ 64px（折叠）
- 支持多级菜单（最多三级）
- 选中菜单项高亮显示（主色调背景+白色文字）
- 支持折叠/展开动画（0.3秒过渡）

#### 5.3.2 顶部导航栏
- 固定高度：60px
- 左侧：系统Logo、面包屑导航
- 右侧：通知图标、用户头像、下拉菜单（包含个人中心、系统设置、退出登录等）

#### 5.3.3 面包屑导航
- 显示当前页面的层级路径
- 支持点击返回上一级页面
- 文字颜色：#606266

## 7. 交互设计

### 7.1 通用交互
- **悬停效果**：按钮、菜单项、链接等元素在鼠标悬停时显示明显的状态变化
- **点击反馈**：按钮点击时有轻微的缩放效果（0.98倍），提供视觉反馈
- **加载状态**：数据加载或操作执行时显示加载动画（使用Element UI的Loading组件）
- **提示信息**：操作成功/失败时显示提示信息（使用Element UI的Message组件）
- **确认对话框**：删除、修改等重要操作前显示确认对话框（使用Element UI的MessageBox组件）

### 7.2 数据交互
- **图表交互**：图表支持鼠标悬停显示详情（tooltip）、点击下钻、缩放等功能
- **表格交互**：表格支持行选中、行展开、排序、筛选等功能
- **表单交互**：表单支持实时验证、焦点切换、自动完成等功能

### 7.3 导航交互
- **左侧导航栏**：支持鼠标悬停展开子菜单、点击切换菜单、折叠/展开导航栏
- **顶部导航栏**：面包屑导航支持点击返回上一级，用户菜单支持下拉展开
- **标签页**：主内容区支持标签页切换，方便用户在多个页面间快速切换

## 10. 动效设计

### 10.1 动效原则
- 动效必须有明确的目的，增强用户体验而非干扰
- 动效应简洁流畅，避免复杂的动画效果
- 动效持续时间应控制在0.1-0.5秒之间
- 同一页面的动效应保持一致的风格和节奏

### 10.2 常用动效
| 动效类型 | 应用场景 | 实现方式 |
|---------|---------|---------|
| 淡入淡出 | 页面切换、弹窗显示/隐藏 | CSS transition: opacity |
| 滑动 | 侧边栏展开/折叠、下拉菜单显示/隐藏 | CSS transition: transform |
| 缩放 | 按钮点击、卡片悬停 | CSS transition: transform |
| 旋转 | 加载动画、刷新按钮 | CSS animation: rotate |
| 渐变 | 骨架屏、加载进度条 | CSS animation: gradient |

### 10.3 动效禁用场景
- 性能较差的设备
- 高对比度模式
- 屏幕阅读器用户
- 用户主动禁用动效

## 11. 无障碍设计

### 11.1 视觉无障碍
- 确保颜色对比度符合WCAG AA标准（文本与背景对比度≥4.5:1）
- 避免仅通过颜色传达信息，应配合图标、文字等
- 支持高对比度模式
- 支持屏幕阅读器

### 11.2 操作无障碍
- 所有功能支持键盘操作（Tab键导航、Enter键确认、Esc键取消等）
- 提供足够大的点击区域（按钮最小44×44px）
- 避免闪烁效果，减少对用户的干扰
- 提供清晰的操作反馈

### 11.3 内容无障碍
- 使用语义化HTML标签
- 为图片、图标等非文本元素提供alt属性
- 保持内容结构清晰，使用适当的标题层级
- 避免使用复杂的表格结构
