# 高铁站节能降耗演示系统说明文档

## 1. 文档说明
本文档详细说明高铁站节能降耗演示系统的架构设计、技术栈、系统功能、部署要求、运行环境和维护指南，为开发人员、运维人员和用户提供系统整体认知和操作指导。

## 2. 系统概述

### 2.1 系统简介
高铁站节能降耗演示系统是一套基于Vue 3+Element UI开发的纯前端演示系统，用于展示高铁站能耗监测、分析、预警和节能管理功能。系统通过模拟真实高铁站能耗数据，提供直观的数据可视化界面，帮助用户了解高铁站能耗状况，制定节能策略，实现降本增效。

### 2.2 系统目标
- 展示高铁站能耗数据的多维度分析能力
- 实现能耗异常的实时监测和预警功能
- 提供节能策略模拟和优化建议
- 展示能耗报告自动生成和分析功能
- 实现用户角色和权限管理

### 2.3 系统特点
- **纯前端实现**：无需后端服务器，直接在浏览器中运行
- **数据持久化**：使用LocalStorage实现数据的本地存储
- **响应式设计**：适配不同屏幕尺寸的设备
- **可视化展示**：采用ECharts实现丰富的数据图表展示
- **模块化设计**：系统功能模块化，便于维护和扩展

## 7. 部署要求

### 7.1 硬件要求
- CPU：双核及以上
- 内存：4GB及以上
- 存储空间：至少2GB可用空间

### 7.2 软件要求
- Web服务器：Nginx 1.16+ 或 Apache 2.4+
- Node.js：16.x LTS及以上
- Git：2.17+（用于代码获取）

### 7.3 网络要求
- 服务器需能访问外部GIT仓库（部署时）
- 客户端需能访问服务器的HTTP/HTTPS端口
- 建议使用稳定的网络环境，带宽不低于10Mbps

## 9. 系统配置

### 9.1 开发环境配置
1. 安装Node.js和npm/yarn
2. 克隆代码仓库：`git clone git@github.com:your-org/energy-saving-demo.git`
3. 安装依赖：`npm install` 或 `yarn install`
4. 启动开发服务器：`npm run dev` 或 `yarn dev`
5. 访问：`http://localhost:5173`

### 9.2 生产环境配置
1. 构建项目：`npm run build` 或 `yarn build`
2. 将`dist`目录部署到Web服务器
3. 配置Web服务器（参考部署文件夹中的GIT部署脚本）

### 9.3 系统参数配置
系统参数配置文件位于`src/config/index.js`，可配置以下参数：
- API接口地址（模拟数据配置）
- 图表默认配置
- 预警阈值默认值
- 系统主题配置

### 10.2 常见问题处理

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 页面加载慢 | 网络问题或资源过大 | 检查网络连接，优化资源加载 |
| 图表不显示 | 数据异常或配置错误 | 检查数据格式，调整图表配置 |
| 功能无响应 | 浏览器兼容性问题 | 升级浏览器或清除缓存 |
| 数据丢失 | 浏览器LocalStorage被清理 | 重新加载页面或导入备份数据 |

### 10.3 日志管理
系统使用浏览器控制台记录日志，可通过以下方式查看：
1. 打开浏览器开发者工具（按F12或Ctrl+Shift+I）
2. 切换到Console标签页查看日志信息
